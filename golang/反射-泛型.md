
# 反射

代码运行中：任意给出一个 函数 或 结构体，能获取该函数的参数  或 结构体内的所有成员变量名 及 变量类型


reflect.TypeOf：获取类型信息
reflect.ValueOf ：获取数据的运行时表示


```go
str := "aaaaa"  
ttDD := reflect.TypeOf(str)  
vvDD := reflect.ValueOf(str)
// string aaaaa
```


打印一个任意结构体的，所有成员变量名及值：

```go
func PrintStruct(mystruct interface{}, separator string) {  
	t := reflect.TypeOf(mystruct)  
	v := reflect.ValueOf(mystruct)  

	for k := 0; k < t.NumField(); k++ {  
		MyPrint(t.Field(k).Name, separator, v.Field(k).Interface())  
	}  
}
```



# 泛型

可以定义一些不确定的类型：any  interface{}
但这些还只是一个变量的类型，如果是：函数的参数也可以定义成泛型，就轻易这么多。
实际上，使用 any interface{} 也可以定义函数的参数，但是需要断言。
这就引出了 GO 的函数泛型实现



```go

// 假设 T 是类型形参，在定义函数时它的类型是不确定的，类似占位符
func Add(a T, b T) T {  
    return a + b
}
// [int]中的 int 是类型实参，代表着函数Add()定义中的类型形参 T 全都被 int 替换
Add[int](100, 200)
//也可以这样写，隐式推导
Add(100, 200)
// 传入类型实参int后，Add()函数的定义可近似看成下面这样：
func Add( a int, b int) int {
    return a + b
}

Add[string]("Hello", "World") 
Add("Hello", "World") 

// 类型实参string传入后，Add()函数的定义可近似视为如下
func Add( a string, b string) string {
    return a + b
}

```

>有点像是 C++ 的 模板

实际参数的类型 T，更像是一个占位符，具体的类型名，由编译的时候，统一替换。


这只是泛型中，函数参数的应用，实际可以应用的更广泛：结构体、数据类型等等。我没在深入研究，感觉泛型还是尽量少用，或者用一些简单的场景 ，因为它是隐式操作，代码阅读就太难了。 


# 底层的具体实现

- 先定位有哪些泛型的函数
- 扫描所有代码，找到调用这个函数的代码
- 分析出，调用方共计使用了哪些参数类型
- 生成全新的函数，如：
- func Add_int、func Add_string、func Add_userStruct、func_point.......
- int string struct 这种比较简单，复杂的是指针类型，它会生成一个通用函数
- func_point(dict)
- dict->一个动态表，保存调用方使用过的结构体

# 泛型对比 interface

- interface 一定会逃逸到堆
- interface 一定得断言，断言所有类型：简单类型+复杂类型的参数
- 泛型对于简单类型可直接使用，不用断言
- 泛型对复杂类型的处理其中跟interface 差不多，也得去动态查找，但好在不用用户来自己断言了，且只需要查找复杂类型即可。
