
# 基础

一个变量分配内存时，申请区域：栈 或 堆
堆：比较大的内存空间，相对松散，且由GC自动回收
栈：连续的，线性的，较小的，有数据结构的。
>每个协程都有一个自己的栈，执行的时候，会直接送到CPU的寄存器执行

栈的申请与释放相对简单：POP PUSH，一个函数执行完毕后，自动释放。
堆：需要另外启动GC，来标记哪些内存是垃圾，之后再统一回收。


一个变量是如何确定分配在：栈 还是 堆 上？
栈：存储函数的 参数、局部变量等，都是相对简单的变量值，容易释放
堆：大的内存，如 object、arrary 等，但并不准确。如果一个变量外部还在引用，那就只能分配到堆上。

分配权：
GO里堆和栈都程序员都无权分配。C/C++里，程序员可以控制堆，不可以控制栈

栈：程序员控制不了，编译器会分析
堆：C语言里使用 malloc  来分配 

# 内存逃逸


```
func A() *int {
    x := 10
    return &x
}
```

x=10，是个简单的成员变量。(正常分配在栈上)
但返回的是个指针，外部调用的函数可能还要继续使用x
GO里，x 被迫需要分到堆上。
并且，当函数执行完毕，正常局部变量都应该释放，但X 就不能释放。


下面3种情况会在堆上分配变量：
- 指针
- interface{}
- 栈空间不足

下面也会在堆上：
1. 函数返回值(局部变量)：不要使用引用/地址
2. 闭包函数，引用外部变量
3. 一个变量过大
4. 指针被存入 slice、map、channel

# 逃逸的变量类型


- interface{}：类型不确定，只能在堆上申请空间
- 闭包，内部函数引用外部变量
- 指针
- channel：内部它有个 queue 存等待队列的，是个指针
- map：内部肯定得有个容器来存KV，也是个指针
- 切片(不确定大小)：因为不确定大小
- 大数组：因为数据太大，一定是会在堆上开空间

以 make 创建的变量类型，返回值本身就是个指针。
map channel array 切片这种，本身就是容器，要存数据 ，存的数据如果是 指针类型，等于连带着都会逃逸的。


# 如何确定逃逸？

```go
// main_pointer.go  
package main  
  
import "fmt"  
  
type Demo struct {  
	name string  
}  
  
func createDemo(name string) *Demo {  
	d := new(Demo) // 局部变量 d 逃逸到堆  
	d.name = name  
	return d  
}  
  
func main() {  
	demo := createDemo("demo")  
	fmt.Println(demo)  
}

```

编译时，通过参数来确定是否逃逸：
```go
go build -gcflags=-m main_pointer.go
```

输出结果为寻找关键字：escapes

# 如何避免?

少用反射、interface、指针等类型
返回值，尽量用值COPY，不返回指针




# 总结

内存逃逸：原本应该分配到栈上的成员变量，因为程序员的失误，被分配到了堆上，增加了性能开销(GC)。并且还有内存泄露的风险。

个人觉得：程序员在内存使用上，要有分寸，虽然有GC帮忙回收，但并不是完全交由GC，程序员要清楚知道哪些变量可能被分配到了堆、栈上，来节约内存，保证程序的健状性。




# 内存泄露


一个变量，其实已经无用了，成为了垃圾，但没有被GC回收，一直占用内存空间。并且，每执行一个函数就会产生一个这样的变量，内存就被占满了。


经常泄露的情况：
- goroutine 泄漏
- map/cache 持有对象
- slice 挂大数组
- channel 堆积
- timer/context 未释放



goroutine ：开启了大量的协程，但都阻塞某个点，函数不能执行完毕，导致协程不结束


slice：

情况1：返回的是个10个字节，但slice底层 是指针，实际大的 slice 并没有被释放
```
func small() []byte {
    big := make([]byte, 1000000)
	//return big[:10]，如果这样返回，GC就无法释放
	
    small := make([]byte, 10)
    copy(small, big[:10])

    return small
}
```

情况2：只要定义一个数组没有确定大小，就是slice，而slice底层就是个指针，就有逃逸和泄露

情况3：基本2 ，只要外部引用不释放，且也没实际作用，如：保存到一个全局变量，那么就是泄露

channel 泄漏 ：其内部会有一个queue  接收 消息，是指针。

map：底层会有buket，是指针。外部调用且不释放，且没用，就是泄露

context :   创建了很多的 context 但不  cancel() 掉。withTimer  也不stop掉
timer：创建了很多的timer ，但不执行：timer.stop  或 timer.C  并没有读出值
